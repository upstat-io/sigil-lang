# E2012: Unknown Capability

A `uses` clause references a capability that doesn't exist.

## Example

```sigil
@fetch (url: str) -> str uses HttpClient = run(    // Error: unknown capability
    // ...
)
```

## Explanation

Capabilities are traits that represent effects. When a function declares
`uses SomeCapability`, that capability must exist as a defined trait.

## Standard Capabilities

| Capability | Purpose |
|-----------|---------|
| `Http` | HTTP client operations |
| `FileSystem` | File I/O operations |
| `Clock` | Time operations |
| `Random` | Random number generation |
| `Cache` | Caching operations |
| `Logger` | Logging operations |
| `Env` | Environment variables |
| `Async` | Async/non-blocking I/O marker |

## Common Causes

1. **Typo in capability name:**
   ```sigil
   // Wrong
   @fetch (url: str) -> str uses HTTP = ...

   // Right
   @fetch (url: str) -> str uses Http = ...
   ```

2. **Missing import:**
   ```sigil
   // Wrong - custom capability not imported
   @process () -> void uses MyCapability = ...

   // Right
   use './capabilities' { MyCapability }
   @process () -> void uses MyCapability = ...
   ```

3. **Capability not defined:**
   ```sigil
   // Need to define custom capabilities first
   trait Database {
       @query (self, sql: str) -> Result<[Row], Error>
       @execute (self, sql: str) -> Result<int, Error>
   }

   @get_users () -> [User] uses Database = ...
   ```

## Solutions

1. **Check spelling:**
   ```sigil
   // Common mistakes
   uses HTTP -> uses Http
   uses Filesystem -> uses FileSystem
   uses Logging -> uses Logger
   ```

2. **Import the capability:**
   ```sigil
   use std.io { FileSystem }
   use std.net { Http }
   ```

3. **Define custom capabilities:**
   ```sigil
   trait EmailService {
       @send (self, to: str, subject: str, body: str) -> Result<void, Error>
   }

   @notify (user: User) -> void uses EmailService = ...
   ```
