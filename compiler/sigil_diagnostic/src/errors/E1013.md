# E1013: function_exp Requires Named Properties

A named-property pattern was called with positional arguments.

## Example

```sigil
@main () -> void = run(
    let result = recurse(n <= 1, 1, n * self(n - 1))    // Error: use named args
)
```

## Explanation

Named-property patterns (`function_exp`) require all arguments to be named.
Unlike regular function calls, these patterns don't accept positional arguments.

Named-property patterns:
- `recurse(condition:, base:, step:, ...)`
- `parallel(tasks:, ...)`
- `spawn(tasks:, ...)`
- `timeout(op:, after:)`
- `cache(key:, op:, ...)`
- `with(acquire:, use:, release:)`
- `for(over:, match:, ...)`

## Common Causes

1. **Positional arguments in pattern:**
   ```sigil
   // Wrong
   recurse(n <= 1, 1, n * self(n - 1))

   // Right
   recurse(
       condition: n <= 1,
       base: 1,
       step: n * self(n - 1)
   )
   ```

2. **Mixed positional and named:**
   ```sigil
   // Wrong
   timeout(slow_op(), after: 5s)

   // Right
   timeout(op: slow_op(), after: 5s)
   ```

## Solutions

1. **Use named arguments:**
   ```sigil
   recurse(
       condition: n <= 1,
       base: 1,
       step: n * self(n - 1)
   )
   ```

2. **For `parallel`:**
   ```sigil
   parallel(
       tasks: [task1, task2, task3],
       max_concurrent: 4
   )
   ```

3. **For `timeout`:**
   ```sigil
   timeout(
       op: slow_operation(),
       after: 30s
   )
   ```

4. **For `cache`:**
   ```sigil
   cache(
       key: user_id,
       op: fetch_user(user_id),
       ttl: 5m
   )
   ```
