# E2011: Named Arguments Required

Function calls with multiple arguments require named arguments.

## Example

```sigil
@add (a: int, b: int) -> int = a + b

@main () -> void = run(
    let sum = add(1, 2)    // Error: multi-arg calls require named arguments
)
```

## Explanation

Sigil requires named arguments for function calls with 2 or more parameters.
This design decision improves code readability and prevents argument-order bugs.

**Exception:** Type conversion functions (`int()`, `float()`, `str()`, `byte()`)
use positional arguments since they have a single, obvious parameter.

## Solutions

1. **Use named arguments:**
   ```sigil
   // Wrong
   let sum = add(1, 2)

   // Right
   let sum = add(a: 1, b: 2)
   ```

2. **Order doesn't matter with named args:**
   ```sigil
   // These are equivalent
   let sum = add(a: 1, b: 2)
   let sum = add(b: 2, a: 1)
   ```

3. **Single-argument functions can be positional:**
   ```sigil
   // Type conversions - positional OK
   let x = int(3.14)
   let s = str(42)

   // Single-param user functions - named required for clarity
   @double (n: int) -> int = n * 2
   let y = double(n: 5)
   ```

## Why Named Arguments?

Consider this call without names:
```sigil
send_email("alice@example.com", "bob@example.com", "Hello", "Meeting tomorrow")
```

With named arguments, the intent is clear:
```sigil
send_email(
    to: "alice@example.com",
    from: "bob@example.com",
    subject: "Hello",
    body: "Meeting tomorrow",
)
```
