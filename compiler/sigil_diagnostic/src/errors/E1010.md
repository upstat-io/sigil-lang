# E1010: Unknown Pattern Argument

An unrecognized argument was passed to a pattern.

## Example

```sigil
@main () -> void = run(
    recurse(
        condition: n <= 1,
        base: 1,
        step: n * self(n - 1),
        foo: 42    // Error: unknown argument 'foo' for pattern 'recurse'
    )
)
```

## Explanation

Built-in patterns accept only specific named arguments. This error occurs when
you pass an argument that the pattern doesn't recognize.

## Valid Arguments by Pattern

### `recurse`
- `condition` (required): When to return base case
- `base` (required): Value to return when condition is true
- `step` (required): Recursive step using `self(...)`
- `memo` (optional): Enable memoization
- `parallel` (optional): Parallelize when value exceeds threshold

### `parallel`
- `tasks` (required): List of tasks to run
- `max_concurrent` (optional): Limit concurrent executions
- `timeout` (optional): Maximum time for all tasks

### `spawn`
- `tasks` (required): List of tasks to fire and forget
- `max_concurrent` (optional): Limit concurrent executions

### `timeout`
- `op` (required): Operation to run
- `after` (required): Duration before timeout

### `cache`
- `key` (required): Cache key
- `op` (required): Operation to cache
- `ttl` (optional): Time-to-live

### `with`
- `acquire` (required): Resource acquisition
- `use` (required): Use the resource
- `release` (optional): Custom cleanup

## Solutions

1. **Remove the unknown argument:**
   ```sigil
   recurse(
       condition: n <= 1,
       base: 1,
       step: n * self(n - 1)
       // removed unknown 'foo' argument
   )
   ```

2. **Check for typos:**
   ```sigil
   // Wrong
   parallel(task: items)

   // Right
   parallel(tasks: items)
   ```

3. **Use the correct pattern:**
   ```sigil
   // If you need memoization, use memo with recurse
   recurse(
       condition: n <= 1,
       base: 1,
       step: self(n - 1) + self(n - 2),
       memo: true
   )
   ```
