# E1007: Missing Function Body

A function declaration has no body expression.

## Example

```sigil
@add (a: int, b: int) -> int    // Error: missing function body
```

## Explanation

Every function in Sigil must have a body. The body is the expression after
the `=` sign that defines what the function returns.

## Common Causes

1. **Forgetting the `= body` part:**
   ```sigil
   // Wrong
   @add (a: int, b: int) -> int

   // Right
   @add (a: int, b: int) -> int = a + b
   ```

2. **Confusing declaration with definition:**
   ```sigil
   // Sigil doesn't have separate declarations - all functions need bodies
   // Wrong: trying to declare without defining
   @process (x: int) -> int

   // Right: always provide a body
   @process (x: int) -> int = x * 2
   ```

## Solutions

1. **Add the function body:**
   ```sigil
   @add (a: int, b: int) -> int = a + b
   ```

2. **For complex functions, use `run`:**
   ```sigil
   @process (items: [int]) -> int = run(
       let filtered = items.filter(predicate: x -> x > 0),
       let sum = filtered.fold(initial: 0, op: (acc, x) -> acc + x),
       sum
   )
   ```

3. **For stub implementations:**
   ```sigil
   // Use panic for not-yet-implemented functions
   @todo (x: int) -> int = panic(msg: "not implemented")
   ```

## Note on Trait Methods

Trait methods can have default implementations or be abstract:

```sigil
trait Printable {
    // Abstract method (no body in trait, must be implemented)
    @to_string (self) -> str

    // Default implementation (has body)
    @print (self) -> void = print(msg: self.to_string())
}
```
