# E1004: Expected Identifier

An identifier was expected but not found.

## Example

```sigil
let 42 = x    // Error: expected identifier, found number
```

## Explanation

Identifiers are names for variables, functions, types, and other declarations.
This error occurs when the parser expects a name but finds something else.

Valid identifiers:
- Start with a letter or underscore
- Contain letters, digits, and underscores
- For functions: start with `@` (e.g., `@main`)
- For config: start with `$` (e.g., `$timeout`)

## Common Causes

1. **Using a literal instead of a name:**
   ```sigil
   // Wrong
   let 123 = x

   // Right
   let x = 123
   ```

2. **Using a keyword as an identifier:**
   ```sigil
   // Wrong - 'type' is a keyword
   let type = "foo"

   // Right
   let type_name = "foo"
   ```

3. **Missing function name:**
   ```sigil
   // Wrong
   @ (x: int) -> int = x + 1

   // Right
   @increment (x: int) -> int = x + 1
   ```

4. **Invalid characters in identifier:**
   ```sigil
   // Wrong
   let my-var = 1

   // Right - use underscores, not hyphens
   let my_var = 1
   ```

## Solutions

1. **Use valid identifier names:**
   ```sigil
   let count = 0
   let _private = 1
   let user_name = "Alice"
   ```

2. **For reserved keywords, use alternative names:**
   ```sigil
   // Keywords: if, then, else, let, mut, type, trait, impl, etc.
   let data_type = "int"
   let for_each = items.map(transform: x -> x * 2)
   ```
