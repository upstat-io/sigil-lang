# E1006: Invalid Function Definition

A function definition is malformed.

## Example

```ori
@add a: int, b: int -> int = a + b    // Error: missing parentheses
```

## Explanation

Function definitions in Ori follow a specific syntax:

```
@name (parameters) -> ReturnType = body
```

This error occurs when the function definition doesn't follow this structure.

## Common Causes

1. **Missing parentheses around parameters:**
   ```ori
   // Wrong
   @add a: int, b: int -> int = a + b

   // Right
   @add (a: int, b: int) -> int = a + b
   ```

2. **Missing arrow (`->`):**
   ```ori
   // Wrong
   @get_value () int = 42

   // Right
   @get_value () -> int = 42
   ```

3. **Missing equals sign:**
   ```ori
   // Wrong
   @get_value () -> int 42

   // Right
   @get_value () -> int = 42
   ```

4. **Wrong order of components:**
   ```ori
   // Wrong
   @add -> int (a: int, b: int) = a + b

   // Right
   @add (a: int, b: int) -> int = a + b
   ```

## Function Syntax

```ori
// Basic function
@name (param: Type) -> ReturnType = expression

// Public function
pub @name (param: Type) -> ReturnType = expression

// Generic function
@name<T> (param: T) -> T = param

// Function with capability
@fetch (url: str) -> str uses Http = http.get(url)

// No-parameter function
@get_value () -> int = 42
```

## Solutions

1. **Use the correct syntax:**
   ```ori
   @add (a: int, b: int) -> int = a + b
   ```

2. **For void functions:**
   ```ori
   @print_hello () -> void = print(msg: "Hello")
   ```
