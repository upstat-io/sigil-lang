# E2020: Unsupported Operator

An operator was used on a type that does not implement the corresponding operator trait.

## Example

```ori
type Point = { x: int, y: int }

@add_points () -> Point = run(
    let a = Point { x: 1, y: 2 },
    let b = Point { x: 3, y: 4 },
    a + b  // Error: Point does not implement Add
)
```

## Explanation

Ori uses operator traits to define operator behavior for user-defined types.
When an operator like `+` is used on a non-primitive type, the compiler looks
for a trait implementation (e.g., `Add` for `+`). If no implementation exists,
this error is emitted.

## Operator Trait Mapping

| Operator | Trait | Method |
|----------|-------|--------|
| `+` | `Add` | `add` |
| `-` | `Sub` | `subtract` |
| `*` | `Mul` | `multiply` |
| `/` | `Div` | `divide` |
| `div` | `FloorDiv` | `floor_divide` |
| `%` | `Rem` | `remainder` |
| `-` (unary) | `Neg` | `negate` |
| `!` (unary) | `Not` | `not` |
| `~` (unary) | `BitNot` | `bit_not` |
| `&` | `BitAnd` | `bit_and` |
| `\|` | `BitOr` | `bit_or` |
| `^` | `BitXor` | `bit_xor` |
| `<<` | `Shl` | `shift_left` |
| `>>` | `Shr` | `shift_right` |

## Solution

Implement the required trait for your type:

```ori
impl Add for Point {
    type Output = Point
    @add (self, rhs: Point) -> Point = Point {
        x: self.x + rhs.x,
        y: self.y + rhs.y,
    }
}
```
