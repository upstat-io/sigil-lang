# E2038: Type Does Not Implement Printable

A value used in string interpolation does not implement the `Printable` trait.

## Example

```ori
type Opaque = { data: int }

let x = Opaque { data: 42 }
let s = `value: {x}`  // ERROR: Opaque does not implement Printable
```

## Explanation

String interpolation (`\`..{expr}..\``) requires that each interpolated
expression has a type implementing `Printable`, which provides the `to_str()`
method used to convert the value to its string representation.

All primitive types (int, float, bool, str, char, byte, Duration, Size,
Ordering) and standard collections ([T], {K: V}, Set<T>, Option<T>,
Result<T, E>, tuples) implement Printable. User-defined types need an
explicit implementation.

## Fix

Add `#derive(Printable)` to the type definition:

```ori
#derive(Printable)
type Opaque = { data: int }
```

Or implement `Printable` manually:

```ori
impl Printable for Opaque {
    @to_str (self) -> str = `Opaque({self.data})`
}
```
