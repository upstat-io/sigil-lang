# E2037: Ambiguous `Into` Conversion

A `.into()` call was made on a type that has multiple `Into<T>`
implementations, and the target type cannot be determined from context.

## Example

```ori
impl Into<str> for UserId {
    @into (self) -> str = self.value
}

impl Into<int> for UserId {
    @into (self) -> int = self.id
}

let x = user_id.into()
//               ^^^^^^ ambiguous: Into<str> or Into<int>?
```

## Explanation

When a type implements `Into<T>` for multiple target types, the compiler
must determine which conversion to use. If the target type cannot be
inferred from context (e.g., a type annotation, function argument type,
or assignment target), the call is ambiguous.

## Solution

Add a type annotation to disambiguate:

```ori
let s: str = user_id.into()  // OK: uses Into<str>
let n: int = user_id.into()  // OK: uses Into<int>
```

Or use the value in a context that constrains the type:

```ori
fn greet (name: str) -> str = `Hello, {name}!`
greet(name: user_id.into())  // OK: parameter type constrains to str
```
