# E1011: Multi-Argument Function Call Requires Named Arguments

A function with multiple parameters was called without using named arguments.

## Example

```ori
@add (a: int, b: int) -> int = a + b

@main () -> void = run(
    let result = add(1, 2)    // Error: use named arguments
)
```

## Explanation

Ori requires named arguments for function calls with multiple parameters.
This prevents errors from argument order confusion and makes code more readable.

The only exceptions are:
- Type conversion functions: `int(x)`, `float(x)`, `str(x)`, `byte(x)`
- Lambda variable calls: parameter names are not known at call site

## Common Causes

1. **Positional arguments in multi-param call:**
   ```ori
   // Wrong
   add(1, 2)

   // Right
   add(a: 1, b: 2)
   ```

2. **Coming from other languages:**
   ```ori
   // Wrong (Python-style)
   send_email("user@example.com", "Hello", "Message body")

   // Right (Ori style)
   send_email(to: "user@example.com", subject: "Hello", body: "Message body")
   ```

## Solutions

1. **Add parameter names:**
   ```ori
   add(a: 1, b: 2)
   ```

2. **Argument order doesn't matter with named arguments:**
   ```ori
   // These are equivalent
   add(a: 1, b: 2)
   add(b: 2, a: 1)
   ```

3. **Type conversions are special (single positional allowed):**
   ```ori
   let n = int("42")      // OK - type conversion
   let s = str(42)        // OK - type conversion
   ```

## Benefits of Named Arguments

- **Self-documenting code:** Intent is clear at call site
- **Order independence:** Arguments can be in any order
- **Refactoring safety:** Adding parameters doesn't break existing calls
- **IDE support:** Better autocomplete and error messages
