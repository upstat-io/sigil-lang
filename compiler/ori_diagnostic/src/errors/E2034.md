# E2034: Invalid Format Specification

A template string interpolation uses a format specification that cannot be parsed.

## Example

```ori
let n = 42
let s = `value: {n:xyz}`
//                 ^^^^ invalid format specification
```

## Explanation

Format specifications follow the syntax:

```
[[fill]align][sign][#][0][width][.precision][type]
```

Where:
- `fill` — any character used for padding (default: space)
- `align` — `<` (left), `^` (center), `>` (right)
- `sign` — `+` (always show), `-` (negative only), ` ` (space for positive)
- `#` — alternate form (e.g., `0x` prefix for hex)
- `0` — zero-pad instead of space-pad
- `width` — minimum field width (integer)
- `.precision` — decimal places for floats, max chars for strings
- `type` — `b` (binary), `o` (octal), `x`/`X` (hex), `e`/`E` (scientific), `f` (fixed), `%` (percent)

## Valid Examples

```ori
let n = 42
let f = 3.14
let s = "hello"

`{n:08x}`       // "0000002a" — zero-padded hex
`{f:.2f}`       // "3.14"     — 2 decimal places
`{s:>10}`       // "     hello" — right-aligned, width 10
`{n:#b}`        // "0b101010" — binary with prefix
```
