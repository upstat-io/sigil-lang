# E1012: Invalid function_seq Syntax

A sequential pattern (`run`, `try`, `match`, `catch`) has invalid syntax.

## Example

```ori
@main () -> void = run(
    let x = 1
    let y = 2    // Error: missing comma between statements
    x + y
)
```

## Explanation

Sequential patterns (`function_seq`) require comma-separated expressions. The
final expression is the return value.

Sequential patterns:
- `run(...)` - Sequential evaluation
- `try(...)` - Error propagation with `?`
- `match(value, arms...)` - Pattern matching
- `catch(expr)` - Catch panics

## Common Causes

1. **Missing commas between statements:**
   ```ori
   // Wrong
   run(
       let x = 1
       let y = 2
       x + y
   )

   // Right
   run(
       let x = 1,
       let y = 2,
       x + y
   )
   ```

2. **Invalid expression in sequence:**
   ```ori
   // Wrong - type definitions can't be in run
   run(
       type Foo = int,
       42
   )
   ```

3. **Missing return expression:**
   ```ori
   // Wrong - run needs a final expression
   run(
       let x = 1,
   )

   // Right
   run(
       let x = 1,
       x
   )
   ```

## Solutions

1. **Separate statements with commas:**
   ```ori
   run(
       let x = compute(),
       let y = process(x),
       combine(x, y)
   )
   ```

2. **For `try`, use `?` for error propagation:**
   ```ori
   try(
       let data = fetch_data()?,
       let parsed = parse(data)?,
       Ok(parsed)
   )
   ```

3. **For `match`, separate arms with commas:**
   ```ori
   match(value,
       Some(x) -> x * 2,
       None -> 0
   )
   ```
