# E2026: Wrong Index Key Type

A subscript expression `x[k]` was used with a key type that does not match
the type's `Index` implementation.

## Example

```ori
type Matrix = { data: [int], cols: int }

impl Index<(int, int), int> for Matrix {
    @index (self, key: (int, int)) -> int = run(
        let (row, col) = key,
        self.data[row * self.cols + col]
    )
}

@get_item () -> int = run(
    let m = Matrix { data: [1, 2, 3, 4], cols: 2 },
    m["hello"]  // Error: expected (int, int), found str
)
```

## Explanation

The type implements `Index` but with a different key type than the one used
in the subscript expression. The key expression must match the `Key` type
parameter of the `Index` implementation.

## Solution

Use the correct key type:

```ori
@get_item () -> int = run(
    let m = Matrix { data: [1, 2, 3, 4], cols: 2 },
    m[(0, 1)]  // Correct: (int, int) key
)
```
