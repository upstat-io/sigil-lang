# E0003: Invalid Number Literal

A number literal is malformed and cannot be parsed.

## Example

```ori
@main () -> void = run(
    let x = 12.34.56    // Error: invalid number literal
)
```

## Explanation

Number literals in Ori must follow a specific format:
- Integers: `42`, `1_000_000`, `0xFF` (hex)
- Floats: `3.14`, `2.5e-8`

Common causes of invalid number literals:
- Multiple decimal points (`12.34.56`)
- Invalid characters in the number (`12abc`)
- Invalid hex format (`0xGG`)
- Leading zeros followed by invalid digits

## Solutions

1. **Fix multiple decimal points:**
   ```ori
   // Wrong
   let x = 12.34.56

   // Right - use a single decimal point
   let x = 12.3456
   ```

2. **Use underscores for readability, not other separators:**
   ```ori
   // Wrong
   let x = 1,000,000

   // Right
   let x = 1_000_000
   ```

3. **Fix hex literals:**
   ```ori
   // Wrong
   let x = 0xZZ

   // Right - hex uses 0-9 and A-F
   let x = 0xFF
   ```
