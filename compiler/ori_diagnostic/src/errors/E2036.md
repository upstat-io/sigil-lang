# E2036: Type Does Not Implement `Into<T>`

A `.into()` call was made on a type that has no `Into` implementation for
the expected target type.

## Example

```ori
type Color = { r: int, g: int, b: int }

let c = Color { r: 255, g: 0, b: 0 }
let s: str = c.into()
//             ^^^^^^ Color does not implement Into<str>
```

## Explanation

The `Into<T>` trait enables explicit, lossless type conversions via the
`.into()` method. Unlike `as`/`as?` (which handle built-in primitive
casts), `Into` is for user-defined semantic conversions.

Standard library implementations:
- `str` -> `Error` (wraps string as error message)
- `int` -> `float` (lossless widening)
- `Set<T>` -> `[T]` (collect set elements to list)

All other conversions require an explicit `impl Into<T> for YourType`.

## Solution

Implement `Into<T>` for the source type:

```ori
impl Into<str> for Color {
    @into (self) -> str = `rgb({self.r}, {self.g}, {self.b})`
}

let c = Color { r: 255, g: 0, b: 0 }
let s: str = c.into()  // OK: "rgb(255, 0, 0)"
```
