# E2033: Trait Cannot Be Derived

A `#[derive(...)]` attribute specified a trait that is not in the set of
automatically derivable traits.

## Example

```ori
#[derive(Iterator)]
type Numbers = { values: [int] }
//     ^^^^^^^^ Iterator cannot be derived
```

## Explanation

Only certain traits support automatic derivation. The compiler can generate
implementations for these traits because they follow a predictable structural
pattern (field-by-field comparison, cloning, hashing, etc.).

Traits with complex or context-dependent behavior cannot be derived and must
be implemented manually.

## Derivable Traits

The following traits can be used with `#[derive(...)]`:

| Trait | Method | Description |
|-------|--------|-------------|
| `Eq` | `eq(self, other: Self) -> bool` | Structural equality |
| `Clone` | `clone(self) -> Self` | Field-by-field cloning |
| `Hashable` | `hash(self) -> int` | Hash computation |
| `Comparable` | `compare(self, other: Self) -> Ordering` | Lexicographic ordering |
| `Printable` | `to_str(self) -> str` | User-facing string representation |
| `Debug` | `debug(self) -> str` | Developer-facing string representation |
| `Default` | `default() -> Self` | Default value construction (structs only) |

## Solution

Implement the trait manually:

```ori
impl Iterator for Numbers {
    type Item = int
    @next ($self) -> Option<int> = ...
}
```
