# E2025: Type Not Indexable

A subscript expression `x[k]` was used on a type that does not implement the `Index` trait.

## Example

```ori
type Config = { name: str, value: int }

@get_item () -> int = run(
    let c = Config { name: "port", value: 8080 },
    c[0]  // Error: Config does not implement Index
)
```

## Explanation

In Ori, the subscript operator `x[k]` works with built-in types (`[T]`, `{K: V}`, `str`)
and any user-defined type that implements the `Index<Key, Value>` trait. If neither
applies, this error is emitted.

## Solution

Implement the `Index` trait for your type:

```ori
impl Index<str, int> for Config {
    @index (self, key: str) -> int = match(key,
        "value" -> self.value,
        _ -> panic("unknown key"),
    )
}
```
