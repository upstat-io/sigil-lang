# E1008: Invalid Pattern Syntax

A pattern is syntactically malformed.

## Example

```ori
@main () -> void = run(
    match(value,
        1 + 2 -> "sum"    // Error: expected pattern, found expression
    )
)
```

## Explanation

Patterns in `match` arms and destructuring bindings must follow pattern syntax,
not expression syntax. Patterns are used to match against values, not compute them.

## Valid Pattern Forms

```ori
42              // Literal pattern
x               // Binding pattern
_               // Wildcard pattern
Some(x)         // Variant pattern
{ x, y }        // Struct pattern
[a, b, ..rest]  // List pattern
1..10           // Range pattern
A | B           // Or-pattern
x @ Some(_)     // At-pattern (bind while matching)
x if guard      // Guard pattern
```

## Common Causes

1. **Using expressions as patterns:**
   ```ori
   // Wrong - expressions are not patterns
   match(value,
       1 + 2 -> "three"
   )

   // Right - use a literal
   match(value,
       3 -> "three"
   )
   ```

2. **Missing variant in destructuring:**
   ```ori
   // Wrong
   match(option,
       x -> use(x)    // Doesn't handle Option properly
   )

   // Right
   match(option,
       Some(x) -> use(x),
       None -> default()
   )
   ```

3. **Invalid syntax in struct pattern:**
   ```ori
   // Wrong
   let { x = 1, y = 2 } = point

   // Right - patterns use : for rename, not =
   let { x: px, y: py } = point
   ```

## Solutions

1. **Use proper pattern syntax:**
   ```ori
   match(result,
       Ok(value) -> handle_success(value),
       Err(e) -> handle_error(e)
   )
   ```

2. **For computed values, bind first:**
   ```ori
   let target = 1 + 2;
   match value {
       x if x == target -> "matched computed value",
       _ -> "other",
   }
   ```
