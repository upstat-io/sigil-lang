# E2035: Format Type Not Supported for Expression Type

A format type character in a template string is incompatible with the
expression's type.

## Example

```ori
let name = "hello"
let s = `{name:08x}`
//              ^^^ hex format requires int, not str
```

## Explanation

Certain format types are only valid for specific value types:

| Format Type | Character | Valid For |
|-------------|-----------|----------|
| Binary | `b` | `int` |
| Octal | `o` | `int` |
| Hex (lower) | `x` | `int` |
| Hex (upper) | `X` | `int` |
| Scientific (lower) | `e` | `float` |
| Scientific (upper) | `E` | `float` |
| Fixed-point | `f` | `float` |
| Percentage | `%` | `float` |

Format specs without a type character (e.g., `{n:>10}`) work with any
type — they only control width, alignment, and fill.

## Solution

Either change the format type to match the value type, or convert the
value first:

```ori
let n = 42
`{n:08x}`             // OK — int with hex format

let f = 3.14
`{f:.2f}`             // OK — float with fixed-point format

let name = "hello"
`{name:>10}`          // OK — no type char, just alignment
```
